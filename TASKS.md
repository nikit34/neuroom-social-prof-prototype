# Задачи на реализацию: Соцдоказательство в сдаче ДЗ

**Гипотеза:** Показ блока «Уже сдали X из Y» при X/Y > Z повышает % сдачи ДЗ в срок.

**Figma:** https://www.figma.com/design/o0b1rcE3DGiNmAv76CWNYd?node-id=13318-29
**Прототип:** `social-proof-homework-submit-prototype/`

## BACK-1: API эндпоинт статистики сдачи ДЗ

**Тип:** Story
**Компонент:** Backend

### Описание
Создать эндпоинт, возвращающий количество сдавших и общее количество получателей для конкретного задания.

### API контракт
```
GET /api/v1/homework/{homework_id}/submit-stats

Response 200:
{
  "submitted": 18,
  "total": 24,
  "ratio": 0.75,
  "recent_submitters": [
    { "initials": "АК", "avatar_color": "#7457FE" },
    { "initials": "МВ", "avatar_color": "#34C759" },
    { "initials": "ДС", "avatar_color": "#FF9500" }
  ]
}
```

### Требования
- `recent_submitters` — последние 3-4 сдавших (для аватарок), без ФИО, только инициалы
- Кэширование ответа на 60 сек (данные не real-time)
- Не возвращать данные если `total < 8` (аудитория слишком мала)
- Учитывать только учеников того же класса, что и текущий пользователь

### Критерии приёмки
- [ ] Эндпоинт возвращает корректные данные для задания с >= 8 получателями
- [ ] Для заданий с < 8 получателями возвращается `204 No Content`
- [ ] `recent_submitters` содержит 3-4 последних сдавших
- [ ] Ответ кэшируется, повторный запрос в пределах 60 сек отдаёт кэш

## BACK-2: Feature flag и конфиг порога показа

**Тип:** Story
**Компонент:** Backend

### Описание
Добавить feature flag `social_proof_hw_submit` и конфигурируемый порог Z для A/B теста.

### Требования
- Feature flag `social_proof_hw_submit` (вкл/выкл)
- Параметр `social_proof_threshold` (float, default `0.6`)
- Поддержка A/B: разделение пользователей на группы с разными порогами (0.5 / 0.6 / 0.75)
- Логирование события `social_proof_shown` с полями: `homework_id`, `user_id`, `ratio`, `threshold`, `variant`

### Критерии приёмки
- [ ] Flag управляется через существующую систему feature flags
- [ ] Порог конфигурируется без деплоя
- [ ] A/B группы назначаются стабильно (один пользователь всегда в одной группе)
- [ ] События логируются в аналитику

## FRONT-1: Компонент SocialProofInline

**Тип:** Story
**Компонент:** Frontend

### Описание
Реализовать компонент соцдоказательства (вариант B — inline с аватарками).

### Дизайн
Два состояния:
- **Default** (ratio 0.6–0.79): фон `#F0EDFF`, текст `#7457FE`, 3 аватарки, текст «X из Y одноклассников уже сдали»
- **High** (ratio >= 0.8): фон `#E8FAF0`, текст `#28A745`, 4 аватарки, текст «X из Y уже сдали — почти все!»

### Пропсы
```ts
interface SocialProofInlineProps {
  submitted: number;
  total: number;
  recentSubmitters: { initials: string; avatarColor: string }[];
}
```

### Требования
- Компонент не рендерится если `submitted/total < threshold` (порог из feature flag)
- Компонент не рендерится если feature flag выключен
- Анимация появления (fade-in + translateY)
- `aria-live="polite"` для accessibility

### Критерии приёмки
- [ ] Компонент рендерит аватарки и текст в соответствии с макетом
- [ ] Переключается между default/high состояниями
- [ ] Не отображается при выключенном flag или ratio ниже порога
- [ ] Анимация при появлении

## FRONT-2: Интеграция в экраны 2.2 и 2.4

**Тип:** Story
**Компонент:** Frontend

### Описание
Встроить `SocialProofInline` в два экрана ученика.

### Точки интеграции
1. **Экран 2.2 «Список ДЗ»** — внутри каждой карточки ДЗ, между описанием и кнопкой
2. **Экран 2.4 «Просмотр домашки»** — между прикреплёнными файлами и кнопкой «Сдать ДЗ на проверку»

### Требования
- Запрос `GET /api/v1/homework/{id}/submit-stats` при маунте карточки/экрана
- Показывать только если до дедлайна <= 3 дней (конфигурируемо)
- Не показывать если ученик уже сдал это ДЗ
- Не дублировать запросы при быстром переключении между карточками

### Критерии приёмки
- [ ] Блок отображается на экране 2.2 в карточках ДЗ
- [ ] Блок отображается на экране 2.4 перед кнопкой CTA
- [ ] Не показывается для уже сданных ДЗ
- [ ] Не показывается если до дедлайна > 3 дней
- [ ] Нет лишних запросов к API

## QA-1: Тест-кейсы для соцдоказательства

**Тип:** Story
**Компонент:** QA

### Функциональные тесты

| # | Сценарий | Ожидаемый результат |
|---|----------|---------------------|
| 1 | ratio > порога, до дедлайна <= 3 дней, total >= 8 | Блок показан |
| 2 | ratio < порога | Блок скрыт |
| 3 | total < 8 | Блок скрыт |
| 4 | До дедлайна > 3 дней | Блок скрыт |
| 5 | Ученик уже сдал ДЗ | Блок скрыт |
| 6 | ratio >= 0.8 | Зелёный стиль, текст «почти все!», 4 аватарки |
| 7 | ratio 0.6–0.79 | Фиолетовый стиль, 3 аватарки |
| 8 | Feature flag выключен | Блок скрыт, запрос к API не уходит |
| 9 | Ученик один в классе (total=1) | API возвращает 204, блок скрыт |
| 10 | Одноклассник сдаёт ДЗ пока экран открыт | После кэш-инвалидации (60 сек) счётчик обновляется |

### Тесты A/B

| # | Сценарий | Ожидаемый результат |
|---|----------|---------------------|
| 11 | Пользователь в группе Z=0.5, ratio=0.55 | Блок показан |
| 12 | Пользователь в группе Z=0.75, ratio=0.55 | Блок скрыт |
| 13 | Один пользователь, два устройства | Одна и та же A/B группа |

### Тесты UI

| # | Сценарий | Ожидаемый результат |
|---|----------|---------------------|
| 14 | Экран 2.2, карточка ДЗ | Блок между описанием и кнопкой |
| 15 | Экран 2.4, просмотр задания | Блок между файлами и «Сдать ДЗ на проверку» |
| 16 | Анимация появления | Плавный fade-in при загрузке данных |
| 17 | Длинный текст предмета / имя | Не ломает верстку |

### Метрики для отслеживания
- `social_proof_shown` — факт показа блока
- `hw_submitted_on_time` — сдача ДЗ до дедлайна
- Сравнение % сдачи в срок: контроль vs варианты A/B
